# Форматы JSON-сообщений от контроллера

### Формат сообщения от контроллера

Сообщения от контроллера передаются в виде JSON и имеют стандартизованный формат. Общий принцип формата состоит в том, что каждое поле именуется 3-значным числовым тегом. Назначение различных тегов описаны ниже в таблицах.

{% hint style="success" %}
Сообщения могут быть инициированы двумя способами:\
\- внутренним событием контроллера;\
\- запросом со стороны сервера.
{% endhint %}

| Код тега |                                           Назначение                                          |
| -------- | :-------------------------------------------------------------------------------------------: |
| 101      |            Идентификатор сообщения на стороне устройства (целый, в пределах Uint32)           |
| 102      |                   Дата и время создания сообщения на устройстве (формат ISO)                  |
| 200      |     Числовой код типа сообщения (см. [таблицу](#tipy-soobshenii-teg-200) кодировки типов)     |
| 300      |           Массив объектов с параметрами сообщения (формат объектов зависит от типа)           |
| 3xx      | Теги, содержащие параметры (см. [таблицу](#tegi-parametrov-soobshenii) с перечнем параметров) |

{% endhint %}

### Теги параметров сообщений

Полный актуальный справочник числовых тегов, используемых в объектах массива `"300"`, вынесен в отдельный документ:

👉 **[`docs/event-property-tags.md`](./event-property-tags.md)** — единственный источник истины по тегам свойств событий.

Краткая справка по основным тегам ячейки и контроллера:

| Код тега | Назначение | Тип |
| -------- | ---------- | --- |
| 301 | Идентификатор (код карты или пинкод) | Строка |
| 302 | Тип устройства ввода. 1 — Wiegand, 4 — TouchPad, 6 — NFC | Enum |
| 303 | Валидация идентификатора: 0 — отсутствует, 1 — привязан, 2 — есть в БД, 3 — Администратор | Enum |
| 304 | Номер двери/ячейки (65535 — не привязан) | Целое 0..65535 |
| 305 | Идентификатор замковой платы | Целое 0..255 |
| 306 | Идентификатор канала замковой платы | Целое 0..255 |
| 307 | Битовое поле состояния замковых датчиков **до** события | Uint32\_t |
| 308 | Битовое поле состояния замковых датчиков **после** события | Uint32\_t |
| 321 | Состояние замка (ячейки, канала): 0 — закрыто, 1 — открыто | Enum\[0,1] |
| 360 | Список состояний ячеек постамата (массив объектов с `304` и `321`) | Массив объектов |

Полный перечень всех тегов см. в [справочнике тегов свойств событий](./event-property-tags.md).
