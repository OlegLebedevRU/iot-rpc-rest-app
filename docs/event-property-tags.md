# Справочник тегов свойств событий (Event Property Tags)

> **Файл:** `docs/event-property-tags.md`  
> **Версия:** 1.0  
> **Дата:** 2026  
> **Статус:** ⭐ Единственный источник истины (canonical / single source of truth)

---

## Назначение

Данный документ является **единым источником истины** по числовым тегам свойств событий, используемых внутри объектов массива `"300"` в MQTT-событиях устройств.

Все остальные документы ссылаются на этот справочник и **не дублируют** полные таблицы тегов.

---

## Формат массива `"300"`

Поле `"300"` является **массивом анонимных JSON-объектов**.

Каждый объект внутри `"300"` состоит из именованных элементов, где:

- **имя элемента** — числовой тег вида `3xx`, `4xx` и т.п., определённый в данном справочнике;
- **значение элемента** — допустимый JSON-тип (`number`, `string`, `boolean`, `object`, `array`, `null`), определяемый спецификацией конкретного тега.

```
"300": [
  {
    "<tag_1>": <value_1>,
    "<tag_2>": <value_2>,
    ...
  },
  ...
]
```

> Поле `"300"` всегда является массивом, даже если содержит один объект.  
> Теги, не определённые в данном справочнике, должны игнорироваться обработчиком.

## Справочник тегов

### Теги идентификатора ввода (`301`–`303`)

| Тег | Ключ | Тип | Описание |
|-----|------|-----|----------|
| `301` | `"301"` | `string` | Идентификатор (код карты, пин-код, биометрия и т.п.) |
| `302` | `"302"` | `int` | Тип устройства ввода: `1` — Wiegand, `4` — TouchPad, `6` — NFC и т.д. |
| `303` | `"303"` | `int` | Результат валидации идентификатора: `0` — не найден, `1` — привязан к ячейке, `2` — есть в БД, но не привязан, `3` — администратор |

### Теги ячейки и контроллера (`304`–`309`)

| Тег | Ключ | Тип | Описание |
|-----|------|-----|----------|
| `304` | `"304"` | `int` | Логический номер ячейки / GPIO / двери (0..65535; `65535` — не привязан) |
| `305` | `"305"` | `int` | Идентификатор (или адрес в сети RS-485) платы подчинённого контроллера (0..255) |
| `306` | `"306"` | `int` | Номер GPIO-канала контроллера (например, номер разъёма на замковой плате) (0..255) |
| `307` | `"307"` | `uint32` | Битовая маска состояния GPIO-датчиков **до** события (по каналам 0–31) |
| `308` | `"308"` | `uint32` | Битовая маска состояния GPIO-датчиков **после** события (по каналам 0–31) |
| `309` | `"309"` | `string` | Список идентификаторов (карт/пинов), связанных с ячейкой (в виде строки, напр., JSON-массива) |

### Теги системной информации устройства (`310`–`336`)

| Тег | Ключ | Тип | Описание |
|-----|------|-----|----------|
| `310` | `"310"` | `string` | Версия ПО/прошивки устройства (напр., `"1.04.025"`) |
| `311` | `"311"` | `int` | Положение DIP-переключателей (битовая маска 16 бит) |
| `312` | `"312"` | `int` | Количество привязанных ячеек к каналам управления |
| `313` | `"313"` | `int` | Свободная память устройства в килобайтах |
| `314` | `"314"` | `string` | MAC-адрес устройства |
| `315` | `"315"` | `uint32` | Битовая маска текущего состояния входов (датчиков) платы |
| `316` | `"316"` | `int` | Количество записей в локальной БД контроллера |
| `317` | `"317"` | `int` | Количество свободных записей в локальной БД |
| `318` | `"318"` | `int` | Номер страницы при постраничной передаче данных (например, БД) |
| `319` | `"319"` | `int` | Признак последней страницы (`1` — последняя, `0` — продолжение) |
| `320` | `"320"` | `array` | Массив объектов с данными (используется при постраничной передаче) |
| `321` | `"321"` | `int` | Состояние канала/ячейки: `0` — закрыто, `1` — открыто |
| `322` | `"322"` | `int` | Идентификатор текущего состояния FSM (Finite State Machine) контроллера |
| `323` | `"323"` | `string` | IP-адрес при подключении по Ethernet |
| `324` | `"324"` | `string` | Серийный номер контроллера, sn |
| `325` | `"325"` | `string` | IP-адрес при подключении по Wi-Fi |
| `326` | `"326"` | `string` | ReplyId — идентификатор принятого серверного сообщения (для подтверждений) |
| `327` | `"327"` | `int` | Состояние кнопок/DIP до изменения (битовая маска 16 бит) |
| `328` | `"328"` | `int` | Состояние кнопок/DIP после изменения (битовая маска 16 бит) |
| `329` | `"329"` | `int` | Версия аппаратной части (Hardware revision) |
| `330` | `"330"` | `int32` | Уникальный числовой ID контроллера в системе |
| `331` | `"331"` | `int` | Тип подключённой платы расширения: `0` — цифровая, `1` — аналоговая |
| `332` | `"332"` | `int` | Идентификатор активной конфигурации приложения (0..16535) |
| `333` | `"333"` | `int` | Идентификатор активной UI-конфигурации (0..16535) |
| `334` | `"334"` | `int` | Номер UART-порта: `0` — UART0, `1` — UART1, `2` — UART2, ... |
| `335` | `"335"` | `string` | Данные с UART в кодировке Base64 |
| `336` | `"336"` | `int32` | Контрольное количество отправленных событий за интервал мониторинга (обычно за 10 минут) |

### Теги состояния ячеек постамата (`360`–`369`)

| Тег | Ключ | Тип | Описание |
|-----|------|-----|----------|
| `360` | `"360"` | `array` | Список состояний ячеек постамата. Массив объектов, каждый из которых содержит теги `304` (номер ячейки) и `321` (состояние ячейки). |
| `361`–`369` | `"361"`..`"369"` | — | Зарезервированы для различных данных о ячейках (будущее использование) |

#### Структура объекта внутри `360`

Каждый элемент массива `"360"` является анонимным объектом, структура которого регулируется данным справочником:

| Тег | Тип | Описание |
|-----|-----|----------|
| `304` | `int` | Логический номер ячейки |
| `321` | `int` | Состояние ячейки: `0` — закрыто, `1` — открыто |

## Примеры

### Пример 1: Открытие ячейки (`200: 13`) с полными данными контроллера

```json
{
  "101": 20338,
  "102": "2026-02-18T01:43:16+03:00",
  "200": 13,
  "300": [
    { "304": 1, "305": 1, "306": 22, "307": 35921924, "308": 8392849 }
  ]
}
```

### Пример 2: Состояние всех ячеек постамата (`200: 63`)

```json
{
  "101": 502,
  "102": "2026-04-22T13:48:02+03:00",
  "200": 63,
  "300": [
    {
      "360": [
        { "304": 1, "321": 1 },
        { "304": 2, "321": 1 },
        { "304": 3, "321": 0 }
      ]
    }
  ]
}
```

> **Интерпретация:** Устройство сообщает о текущем состоянии всех ячеек: ячейки №1 и №2 открыты (`321: 1`), ячейка №3 закрыта (`321: 0`).
