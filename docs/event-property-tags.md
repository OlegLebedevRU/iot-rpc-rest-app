# Справочник тегов свойств событий (Event Property Tags)

> **Файл:** `docs/event-property-tags.md`  
> **Версия:** 1.0  
> **Дата:** 2026  
> **Статус:** ⭐ Единственный источник истины (canonical / single source of truth)

---

## Назначение

Данный документ является **единым источником истины** по числовым тегам свойств событий, используемых внутри массива `"300"` в MQTT-событиях `dev/<SN>/evt`.

Все остальные документы ссылаются на этот справочник и **не дублируют** полные таблицы тегов.

---

## Формат массива `"300"`

Поле `"300"` является **массивом анонимных JSON-объектов**.

Каждый объект внутри `"300"` состоит из именованных элементов, где:

- **имя элемента** — числовой тег вида `3xx`, `4xx` и т.п., определённый в данном справочнике;
- **значение элемента** — допустимый JSON-тип (`number`, `string`, `boolean`, `object`, `array`, `null`), определяемый спецификацией конкретного тега.

```json
"300": [
  {
    "304": 1,
    "321": 1
  }
]
```

> Поле `"300"` всегда является массивом, даже если содержит один объект.  
> Теги, не определённые в данном справочнике, должны игнорироваться обработчиком.

| Тег | Тип | Описание | Platerra | Siplite |
|-----|-----|----------|:--------:|:-------:|
| `301` | `string` | Идентификатор (код карты, пин-код, биометрия и т.п.) | — | ✅ |

#### Структура объекта внутри `360`

Каждый элемент массива `"360"` является анонимным объектом, структура которого регулируется данным справочником.

| Тег | Тип | Описание |
|-----|-----|----------|
| `304` | `int` | Логический номер ячейки |
| `321` | `int` | Состояние ячейки: `0` — закрыто, `1` — открыто |

#### Структура объекта `370` (`seed_data`)

Тег `370` используется в событии `SeedInitDone` (`200 = 65`) и содержит **полный snapshot seed-flow**.

| Подполе | Тип | Обязат. | Описание |
|---|---|---|---|
| `ver` | `uint32` | да | Версия схемы seed (`cfg/cfg_init/schema_version`). |
| `reason` | `string` | да | Причина seed: `first_boot`, `factory_reset`, `schema_upgrade` (зарезервировано на будущее). |
| `applied_at` | `string` | нет | ISO-8601 с offset — момент завершения seed. Если отсутствует, следует использовать `102` события. |
| `device.fw` | `string` | да | Версия прошивки, согласованная с тегом `310` в `Hello`/`DevHelthCheck`. |
| `device.model` | `string` | да | Модель платы / устройства. |
| `device.mac1` | `string` | да | MAC-адрес первого порта / интерфейса устройства. |
| `device.mac2` | `string` | да | MAC-адрес второго порта / интерфейса устройства. |
| `records` | `array<RecordObj>` | да | Полный snapshot seed-записей. Каждый элемент имеет формат `{p, ns, k, t, v}`. |
| `truncated` | `boolean` | нет | Флаг, что список `records[]` был усечён из-за ограничения размера payload. |

#### Формат элементов `records[]` внутри `370`

Каждый объект массива `records` должен быть совместим по семантике с контрактом NVS-операций (`method_code = 49` / `50`):

| Поле | Тип | Описание |
|-----|-----|----------|
| `p` | `string` | Партиция NVS, например `cfg` или `db1` |
| `ns` | `string` | Namespace записи |
| `k` | `string` | Ключ записи |
| `t` | `string` | Тип значения (`str`, `i32`, `u8` и т.п.) |
| `v` | `any JSON type` | Значение записи |

## Примеры

### Пример 1: Открытие ячейки (`200: 13`) с полными данными контроллера

```json
{
  "101": 20338,
  "102": "2026-02-18T01:43:16+03:00",
  "200": 13,
  "300": [
    { "304": 1, "305": 1, "306": 22, "307": 35921924, "308": 8392849 }
  ]
}
```

### Пример 2: Состояние всех ячеек постамата (`200: 63`)

```json
{
  "101": 502,
  "102": "2026-04-22T13:48:02+03:00",
  "200": 63,
  "300": [
    {
      "360": [
        { "304": 1, "321": 1 },
        { "304": 2, "321": 1 },
        { "304": 3, "321": 0 }
      ]
    }
  ]
}
```

> **Интерпретация:** Устройство сообщает о текущем состоянии всех ячеек: ячейки №1 и №2 открыты (`321: 1`), ячейка №3 закрыта (`321: 0`).

### Пример 3: Seed snapshot (`200: 65`, тег `370`)

```json
{
  "101": 17,
  "102": "2026-04-26T12:00:00+03:00",
  "200": 65,
  "300": [
    {
      "370": {
        "ver": 1,
        "reason": "first_boot",
        "applied_at": "2026-04-26T11:59:58+03:00",
        "device": {
          "fw": "1.4.025",
          "model": "leo4-sip-v1.0",
          "mac1": "cc:db:a7:1e:56:4b",
          "mac2": "cc:db:a7:1e:56:4c"
        },
        "records": [
          {"p": "cfg", "ns": "cfg_sip", "k": "sip_uri", "t": "str", "v": "udp://6002:6002@87.242.100.34:5060"},
          {"p": "cfg", "ns": "cfg_sip", "k": "reg_interval", "t": "i32", "v": 2700},
          {"p": "db1", "ns": "auth_sip", "k": "7777", "t": "str", "v": "7777"}
        ]
      }
    }
  ]
}
```

## Диапазоны тегов

| Диапазон | Назначение |
|---------|-----------|
| `301`–`303` | Данные события ввода идентификатора (`IdInput`) |
| `304`–`309` | Данные о ячейках, каналах и GPIO |
| `310`–`340` | Системные данные устройства (Hello/HealthCheck), DIP, SIP-диагностика |
| `341`–`346` | Результаты OTA и загрузки бинарника STM32 |
| `347`–`352` | Результаты прошивки STM32 |
| `353`–`354` | SIP-регистрация |
| `355`–`359` | Зарезервированы |
| `360` | Список состояний ячеек постамата |
| `361`–`369` | Зарезервированы для данных о ячейках постамата |
| `370` | `seed_data` — snapshot seed-flow для `SeedInitDone` (`200 = 65`) |
| `371`–`398` | Зарезервированы |
| `399` | Raw payload (внутренний passthrough) |

---

## Совместимость и расширяемость

- **Клиентам рекомендуется игнорировать неизвестные теги** — новые теги могут добавляться в любой диапазон без нарушения обратной совместимости.
- Зарезервированные теги (`355`–`359`, `361`–`369`, `371`–`398`) **не должны использоваться** клиентами для хранения или обработки собственных данных.
- Тег `370` закреплён за структурой `seed_data` и не должен переиспользоваться для иной семантики.
- Тег `399` является внутренним механизмом кодека для событий с `msg_event_type >= 1000` и не является частью стандартного публичного API.
- Один и тот же тег (например, `304`) может иметь одинаковый семантический смысл в разных событиях — это является нормой и определяется типом события `200`.
