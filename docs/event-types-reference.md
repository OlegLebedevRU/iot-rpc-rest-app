# Справочник MQTT-событий (evt) и типов (тег 200)

> **Файл:** `docs/event-types-reference.md`
> **Назначение:** Референсный документ (источник истины) по типам событий (поле `200`), отправляемым устройством в MQTT-топик `dev/<SN>/evt`.
> **Важно:** В этом файле не приводится расшифровка назначения кодов объектов внутри массива `300` (теги 3xx). Все 3xx-теги и их типы описаны в [`docs/event-property-tags.md`](./event-property-tags.md).

---

## 1. Общий формат события

Все события от устройства (в топик `dev/<SN>/evt`) публикуются в виде JSON-объекта со следующей базовой структурой:

```json
{
  "101": 12345,
  "102": "2026-04-22T13:48:02+03:00",
  "200": 0,
  "300": [
    { ... }
  ]
}
```

- **`101`** (uint32): Уникальный идентификатор события на устройстве (в пределах сессии/аптайма).
- **`102`** (string): Временная метка ISO 8601 (с часовым поясом).
- **`200`** (int): Код типа события (см. таблицу ниже). Полностью определяет структуру вложенных параметров.
- **`300`** (array): Массив объектов с параметрами события. Структура содержимого зависит от типа `200`.

---

## 2. Справочник-таблица типов событий (Код 200)

| Код (`200`) | Название / Тип | Описание |
| :--- | :--- | :--- |
| **0** | `Hello` | Автоматическая отправка при первой инициализации устройства (после `MQTT_EVENT_CONNECTED`). |
| **2** | `DoorSensorState` | Изменение состояния GPIO-входов (например, массив замков/дверей открыт или закрыт). |
| **3** | `IdInput` | Пользователь ввел идентификатор (RFID-карта, PIN-код, биометрия). |
| **13** | `CellOpenEvent` | Событие GPIO/Входа: открытие конкретной ячейки. |
| **14** | `CellCloseEvent` | Событие GPIO/Входа: закрытие конкретной ячейки. |
| **23** | `CellBindedAct` | Привязка логического номера ячейки к аппаратному каналу (hardware контроллеру). |
| **27** | `OtaResult` | Результат обновления прошивки ESP32 (OTA_OK, ошибки). |
| **38** | `BoardConnected` | В системе обнаружена (подключена) плата расширения. |
| **39** | `BoardDisconnected`| Плата расширения отключена (потеряна связь). |
| **44** | `DevHelthCheck` | Регулярный системный пинг/статус от устройства (keep-alive). |
| **46** | `UartToCloud` | Входные данные с внешнего порта в конфигурации «Шлюз UART -> MQTT». |
| **53** | `Stm32FwUpdateResult`| Результат прошивки сопроцессора STM32 по локальной шине. |
| **54** | `SipRegStateChange`| Факт входа или выхода из состояния SIP `REGISTERED` (с фильтрацией штатных переподключений). |
| **63** | `FullStateNotify` | Текущее бинарное состояние всех ячеек (слотов, внешних GPIO и т.п.). |
| **65** | `SeedInitDone` | Однократное событие завершения seed-flow после first boot или factory reset. Содержит снимок применённых seed-записей в теге `370` (`seed_data`). |

---

## 3. Полные форматы известных событий

### Код 0: `Hello`
Первичная инициализация устройства.
```json
{
  "101": 1,
  "102": "2026-04-22T10:00:00+00:00",
  "200": 0,
  "300": [
    {
      "310": "1.05.000",
      "311": 15,
      "314": "aa:bb:cc:dd:ee:ff",
      "323": "192.168.1.50",
      "324": "sn12345678",
      "340": { "state": "SIP_IDLE", "diag": { "cnt": {}, "cf": 0, "crit": [] } }
    }
  ]
}
```

### Код 3: `IdInput`
Срабатывание устройства ввода (например, RFID).
```json
{
  "101": 36823,
  "102": "2026-03-03T12:41:33+03:00",
  "200": 3,
  "300": [
    {
      "301": "044AFE42C76781",
      "302": 6,
      "303": 0
    }
  ]
}
```

### Коды 13 / 14: `CellOpenEvent` / `CellCloseEvent`
Событие открытия (13) или закрытия (14) ячейки.
```json
{
  "101": 20338,
  "102": "2026-02-18T01:43:16+03:00",
  "200": 13,
  "300": [
    {
      "304": 12,
      "305": 1,
      "306": 22,
      "307": 35921924,
      "308": 8392849
    }
  ]
}
```

### Код 27: `OtaResult`
Результат загрузки и применения OTA-обновления.
```json
{
  "101": 1500,
  "102": "2026-04-20T11:00:00+03:00",
  "200": 27,
  "300": [
    {
      "341": "OTA_OK",
      "342": "Update successful"
    }
  ]
}
```

### Код 44: `DevHelthCheck`
Регулярный пинг/статус устройства. Формат аналогичен `Hello` (Код 0).
```json
{
  "101": 1047,
  "102": "2026-08-12T10:31:57+00:00",
  "200": 44,
  "300": [
    {
      "310": "1.04.025",
      "313": 65740,
      "324": "a3b1234567c10221d290825",
      "339": "3600",
      "340": { "state": "SIP_REGISTERED", "diag": { "cnt": {}, "cf": 0, "crit": [] } }
    }
  ]
}
```

### Код 54: `SipRegStateChange`
Изменение состояния SIP-субсистемы устройства.
```json
{
  "101": 20456,
  "102": "2026-04-02T14:23:45+03:00",
  "200": 54,
  "300": [
    {
      "353": "LOST_REGISTERED",
      "354": "ip_lost",
      "340": { "state": "SIP_UNREGISTERED", "diag": { "cnt": {}, "cf": 1, "crit": [] } }
    }
  ]
}
```

### Код 63: `FullStateNotify` (Текущее бинарное состояние всех ячеек)
Новое событие для передачи полного слепка состояния (открыто/закрыто, активно/неактивно) всех подконтрольных ячеек.
```json
{
  "101": 502,
  "102": "2026-04-22T13:48:02+03:00",
  "200": 63,
  "300": [
    {
      "360": [
        { "304": 1, "321": 1 },
        { "304": 2, "321": 1 },
        { "304": 3, "321": 0 }
      ]
    }
  ]
}
```

### Код 65: `SeedInitDone`
Однократное событие завершения seed-flow. Устройство отправляет его после успешного применения seed-записей и после установления MQTT-соединения. Для повторных обычных reboot событие не отправляется. Все данные snapshot упакованы в тег `370` (`seed_data`).

```json
{
  "101": 17,
  "102": "2026-04-26T12:00:00+03:00",
  "200": 65,
  "300": [
    {
      "370": {
        "ver": 1,
        "reason": "first_boot",
        "applied_at": "2026-04-26T11:59:58+03:00",
        "device": {
          "fw": "1.4.025",
          "model": "leo4-sip-v1.0",
          "mac1": "cc:db:a7:1e:56:4b",
          "mac2": "cc:db:a7:1e:56:4c"
        },
        "records": [
          {"p": "cfg", "ns": "cfg_sip", "k": "sip_uri", "t": "str", "v": "udp://6002:6002@87.242.100.34:5060"},
          {"p": "cfg", "ns": "cfg_sip", "k": "reg_interval", "t": "i32", "v": 2700},
          {"p": "cfg", "ns": "cfg_sip", "k": "keepalive", "t": "i32", "v": 30},
          {"p": "cfg", "ns": "cfg_eth", "k": "eth_ip", "t": "str", "v": "192.168.77.1"},
          {"p": "cfg", "ns": "cfg_mqtt", "k": "broker_host", "t": "str", "v": "dev.leo4.ru"},
          {"p": "cfg", "ns": "cfg_frontend", "k": "wiegand", "t": "str", "v": "{\"auth_ns\":\"auth_door\",\"flags\":124,\"script\":\"open_door\",\"fallback\":\"free_pass\"}"},
          {"p": "cfg", "ns": "cfg_frontend", "k": "touch_keypad", "t": "str", "v": "{\"auth_ns\":\"auth_sip\",\"flags\":124,\"script\":\"sip_map\"}"},
          {"p": "db1", "ns": "auth_admin", "k": "993301", "t": "str", "v": "display_ip"},
          {"p": "db1", "ns": "auth_sip", "k": "7777", "t": "str", "v": "7777"}
        ]
      }
    }
  ]
}
```

> **Примечания по совместимости:**
> - Код события `65` зарезервирован под `SeedInitDone`; код `64` не используется для seed-flow.
> - Клиенты, не знающие `200 = 65`, должны игнорировать сообщение по стандартному правилу forward compatibility.
> - Формат элементов `records[]` согласован с контрактом записи/чтения NVS через `method_code = 49` / `50`.
