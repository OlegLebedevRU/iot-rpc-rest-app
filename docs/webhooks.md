# 🌐 Вебхуки — Интеграция с внешними системами

Данный документ описывает, как настроить и использовать вебхуки для получения событий от сервера в вашу систему.  
Все события (например, команды, результаты задач) отправляются через HTTP-запросы на указанный URL.

---

## 🔑 Требования к доступу

- Все запросы к API должны содержать заголовок:
  ````http 
  x-api-key: ваш_секретный_ключ


- Ключ привязан к организации и выдаётся администратором.
- `org_id` автоматически определяется по ключу — не нужно передавать его в URL.

---

## 📦 Поддерживаемые типы событий


| Тип события         | Описание |
|---------------------|--------|
| `msg-event`         | События с устройства (например, открытие двери, сканирование QR) |
| `msg-task-result`   | Результат выполнения задачи (например, ответ на команду открытия) |

---

## 🛠 API управления вебхуками

Базовый путь: `/api/v1/webhooks`

> ⚠️ Все методы требуют `x-api-key`.

### 1. Получить все вебхуки
```http 
GET /api/v1/webhooks/
```

**Ответ:**
````json 
[ { "org_id": 100, "event_type": "msg-event", "url": "https://your-api.com/events", "headers": {"Authorization": "Bearer abc123"}, "is_active": true, "created_at": "2025-04-05T10:00:00+00:00", "updated_at": "2025-04-05T10:05:00+00:00" } ]
````

---

### 2. Установить или обновить вебхук

```http 
PUT /api/v1/webhooks/{event_type}
```

**Параметры пути:**
- `event_type`: `msg-event` или `msg-task-result`

**Тело запроса:**
```json 
{ "url": "https://your-api.com/webhook/events", "headers": { "Authorization": "Bearer abc123", "X-Custom-Header": "value" }, "is_active": true }
```

**Пример запроса:**
```http 
PUT /api/v1/webhooks/msg-event Content-Type: application/json x-api-key: your-secret-key
{ "url": "https://your-api.com/events", "headers": {"Authorization": "Bearer abc123"} }
```

**Ограничения:**
- Максимум **2 вебхука на организацию** (настраивается).
- URL должен быть валидным (`http://` или `https://`).
- Сервер проверит доступность URL при сохранении.

**Успешный ответ:**
```json
 { "org_id": 100, "event_type": "msg-event", "url": "https://your-api.com/events", "headers": {"Authorization": "Bearer abc123"}, "is_active": true, "created_at": "2025-04-05T10:00:00+00:00", "updated_at": null }
```

---

### 3. Удалить вебхук
```http 
DELETE /api/v1/webhooks/{event_type}
```

**Пример:**
```http 
DELETE /api/v1/webhooks/msg-task-result x-api-key: your-secret-key
```

**Ответ:**
```json 
{ "message": "Webhook deleted" }
```

## 📤 Формат входящих вебхуков

Когда происходит событие, ваш сервер получает `POST`-запрос.

### Общие заголовки

| Заголовок        | Пример значение       | Описание |
|------------------|-----------------------|--------|
| `x-msg-type`     | `msg-event`           | Тип сообщения |
| `x-device-id`    | `12345`               | ID устройства |
| `Content-Type`   | `application/json`    | Всегда JSON |

> Для `msg-task-result` дополнительно:
> - `x-ext-id`: внешний ID задачи (если был указан)
> - `x-result-id`: внутренний ID результата

### Пример тела (`msg-event`)
```json
{
  "102":"2026-03-11T11:25:52ZMSK",
  "200":3,
  "300":[{"301":"044AFE42C76781","302":6,"303":0}],
  "101":48278
}
```

### Пример тела (`msg-task-result`)
```json 
{ 
  "task_id": "a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8",
  "device_id": 12345, 
  "status": "success", 
  "method_code": 100, 
  "result": "Door opened", 
  "timestamp": "2025-04-05T10:01:00Z"
}
```

## ⚙️ Настройки вебхуков (через `.env`)

| Переменная | По умолчанию | Описание |
|-----------|-------------|--------|
| `APP_CONFIG__WEBHOOK__MAX_PER_ORG` | `2` | Макс. количество вебхуков на организацию |
| `APP_CONFIG__WEBHOOK__TIMEOUT` | `5.0` | Таймаут при проверке доступности URL (сек) |
| `APP_CONFIG__WEBHOOK__MAX_RETRIES` | `3` | Кол-во попыток отправки при ошибке (в будущем) |

---

## ❗ Ошибки

| Код | Описание |
|-----|--------|
| `400 Bad Request` | Неверный `event_type`, URL или превышен лимит |
| `403 Forbidden` | Неверный или отсутствующий `x-api-key` |
| `404 Not Found` | Вебхук не найден при удалении |
| `422 Unprocessable Entity` | Невалидный JSON или структура |

---

## 📝 Примечания

- При создании/обновлении вебхука сервер делает **GET-запрос** к URL для проверки доступности.
- Если URL недоступен — вебхук не будет сохранён.
- Отправка событий происходит асинхронно.
- Поддержка повторных попыток будет добавлена в будущих версиях.
