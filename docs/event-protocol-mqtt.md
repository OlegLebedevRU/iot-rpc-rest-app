# Протокол MQTT-сообщений для отправки событий (evt)

> **Файл:** `docs/event-protocol-mqtt.md`  
> **Версия:** 0.40  
> **Дата:** 2026  
> **Тема:** Формат асинхронных событий в топике `dev/<SN>/evt`

---

## Назначение

Данный документ описывает формат **MQTT-сообщений для отправки асинхронных событий** от устройства (приложения или контроллера) в облако через топик:

где `<SN>` — серийный номер устройства (уникальный идентификатор).

Пример sn: **a3b1234567c10221d290825**

События (`evt`) используются для передачи информации о внутренних изменениях состояния устройства: срабатывание датчиков, действия пользователя, системные уведомления и пр.

---

## Общая структура сообщения

Сообщение представляет собой **JSON-объект**, содержащий поля с числовыми тегами. Поддерживается единый формат для всех типов событий.

### Основные поля:

| Тег | Ключ | Тип | Описание |
|-----|------|-----|--------|
| `101` | `"101"` | `uint32` | Уникальный идентификатор события на стороне устройства (в пределах сессии или перезагрузки) |
| `102` | `"102"` | `string` | Временная метка создания события в формате ISO 8601. Может содержать часовой пояс (например, `ZMSK` для MSK). |
| `200` | `"200"` | `int` | Код типа события (см. таблицу ниже) |
| `300` | `"300"` | `array` | Массив объектов с параметрами события. Структура зависит от значения `200`. |

> Поле `300` всегда является массивом, даже если содержит один объект.

## Примеры сообщений

### Пример 1: Событие "Датчик замка (открытие)" (`200: 13`)
```json 
{
    "101": 20338,
    "300": [{ "304": 65535, "306": 22, "305": 1, "307": 35921924, "308": 8392849 }],
    "102": "2026-02-18T01:43:16ZMSK",
    "200": 13
}
```

### Пример 2: Событие "Ввод идентификатора (карта, пин и т.п.)" (`200: 3`)
````json 
{ "101": 36823, "102": "2026-03-03T12:41:33ZMSK", "200": 3, "300": [ { "301": "044AFE42C76781", "302": 6, "303": 0 } ] }

````
> **Интерпретация:** Пользователь ввёл идентификатор (например, карту), который не найден в локальной базе (`303: 0`). Источник — устройство ввода типа 6 (NFC считыватель).

## Коды типов событий (тег `200`)

| Код (`200`) | Название | Описание                                                                                        | Параметры (`3xx`)                   |
|------------|---------|-------------------------------------------------------------------------------------------------|-------------------------------------|
| `0` | `Hello` | Первичная инициализация устройства                                                              | `310`, `311`, `312`, `313`, `314`   |
| `2` | `DoorSensorState` | Состояние GPIO-входов контроллера (например, замковый контроллер - замки ячеек открыты/закрыты) |                                     | `305`, `315` |
| `3` | `IdInput` | Событие ввод идентификатора (карта, пин-код)                                                    | `301`, `302`, `303`                 |
| `13` | `CellOpenEvent` | Событие GPIO (вход), например "Открытие ячейки"                                                 | `304`, `305`, `306`, `307`, `308`   |
| `14` | `CellCloseEvent` | Событие GPIO (вход), например  "Закрытие ячейки"                                                | `304`, `305`, `306`, `307`, `308`   |
| `23` | `CellBindedAct` | Привязка логического номера ячейки к hardware (контроллеру)                                     | `304`, `305`, `306`                 |
| `38` | `BoardConnected` | Обнаружена плата расширения                                                                     | `305`, `315`                        |
| `39` | `BoardDisconnected` | Отключена (отвалилась) плата расширения                                                         | `305`, `315`                        |
| `44` | `DevHelthCheck` | Регулярный системный пинг от устройства                                                         | Аналогично `Hello` + доп... |
| `46` | `UartToCloud` | Входные данные с UART-порта в конфигурации "Шлюз UART->MQTT"                                    | `334`, `335`                        |

## Теги параметров сообщений

Ниже представлена таблица параметров (тегов), используемых в массиве `"300"` (дополняется).

| Тег | Ключ | Тип данных | Описание                                                                                                                           |
|-----|------|------------|------------------------------------------------------------------------------------------------------------------------------------|
| `301` | `"301"` | `string` | Идентификатор (код карты, пин-код, биометрия и т.п.)                                                                               |
| `302` | `"302"` | `int` | Тип устройства ввода: `1` — Wiegand, `4` — TouchPad, `6` — NFC и т.д.                                                              |
| `303` | `"303"` | `int` | Результат валидации идентификатора: `0` — не найден, `1` — привязан к ячейке, `2` — есть в БД, но не привязан, `3` — администратор |
| `304` | `"304"` | `int` | Логический номер (number, cell_number, num, gpio_num, ...) GPIO/ячейки/двери (0..65535; 65535 — не привязан)                       |
| `305` | `"305"` | `int` | Идентификатор (или адрес в сети RS-485) платы подчиненного контроллера (0..255)                                                    |
| `306` | `"306"` | `int` | Номер GPIO-канала контроллера (например, номер разъема на замковой плате) (0..255)                                                 |
| `307` | `"307"` | `uint32` | Битовая маска состояния GPIO-датчиков **до** события (по каналам 0–31)                                                             |
| `308` | `"308"` | `uint32` | Битовая маска состояния GPIO-датчиков **после** события (по каналам 0–31)                                                          |
| `309` | `"309"` | `string` | Список идентификаторов (карт/пинов), связанных с ячейкой (в виде строки, напр., JSON-массива)                                      |

| `312` | `"312"` | `int` | Количество привязанных ячеек к каналам управления                                                                                  |
| `313` | `"313"` | `int` | Свободная память устройства в килобайтах                                                                                           |
| `314` | `"314"` | `string` | MAC-адрес устройства                                                                                                               |
| `315` | `"315"` | `uint32` | Битовая маска текущего состояния входов (датчиков) платы                                                                           |
| `316` | `"316"` | `int` | Количество записей в локальной БД контроллера                                                                                      |
| `317` | `"317"` | `int` | Количество свободных записей в локальной БД                                                                                        |
| `318` | `"318"` | `int` | Номер страницы при постраничной передаче данных (например, БД)                                                                     |
| `319` | `"319"` | `int` | Признак последней страницы (`1` — последняя, `0` — продолжение)                                                                    |
| `320` | `"320"` | `array` | Массив объектов с данными (используется при постраничной передаче)                                                                 |
| `321` | `"321"` | `int` | Состояние канала/ячейки: `0` — закрыто, `1` — открыто                                                                              |
| `322` | `"322"` | `int` | Идентификатор текущего состояния FSM (Finite State Machine) контроллера                                                            |
| `323` | `"323"` | `string` | IP-адрес при подключении по Ethernet                                                                                               |
| `324` | `"324"` | `string` | Серийный номер контроллера                                                                                                         |
| `325` | `"325"` | `string` | IP-адрес при подключении по Wi-Fi                                                                                                  |
| `326` | `"326"` | `string` | ReplyId — идентификатор принятого серверного сообщения (для подтверждений)                                                         |
| `327` | `"327"` | `int` | Состояние кнопок/DIP до изменения (битовая маска 16 бит)                                                                           |
| `328` | `"328"` | `int` | Состояние кнопок/DIP после изменения (битовая маска 16 бит)                                                                        |
| `329` | `"329"` | `int` | Версия аппаратной части (Hardware revision)                                                                                        |
| `330` | `"330"` | `int32` | Уникальный числовой ID контроллера в системе                                                                                       |
| `331` | `"331"` | `int` | Тип подключенной платы расширения: `0` — цифровая, `1` — аналоговая                                                                |
| `332` | `"332"` | `int` | Идентификатор активной конфигурации приложения (0..16535)                                                                          |
| `333` | `"333"` | `int` | Идентификатор активной UI-конфигурации (0..16535)                                                                                  |
| `334` | `"334"` | `int` | Номер UART-порта: `0` — UART0, `1` — UART1, `2` — UART2, ...                                                                       |
| `335` | `"335"` | `string` | Данные с UART в кодировке Base64                                                                                                   |
| `336` | `"336"` | `int32` | Контрольное количество отправленных событий за интервал мониторинга (обычно за 10 минут)                                           |

## Передача по MQTT

### Публикация события

Устройство публикует сообщение в топик:
dev/<SN>/evt
#### Пример:
```json
- Topic: dev/a3b1234567c10221d290825/evt 
- QoS: 1 
- Retain: 0
- Payload:         
{ "101": 36823, "102": "2026-03-03T12:41:33ZMSK", "200": 3, "300": [ { "301": "044AFE42C76781", "302": 6, "303": 0 } ] }
- User Properties:
  -- event_type_code: 3
  -- dev_event_id: 36823
  -- dev_timestamp: 1740984093 
  -- correlation_id: d9afcbfa-3d2c-4304-8e69-f644fef29f1f
```


> **Рекомендации:**
> - Использовать **QoS 1** для гарантированной доставки.
> - Заполнять `User Properties` для фильтрации на сервере.
> - Генерировать уникальный `correlation_id` (UUID4) для каждого события.

## Рекомендации по реализации

- Все временные метки (`101`) должны быть в UTC или с явным указанием зоны (`ZMSK`, `+03:00` и т.п.).
- Идентификатор `101` должен быть уникален в пределах активной сессии устройства.
- Сообщения должны быть валидными JSON без лишних пробелов (для экономии трафика).
- При отправке через медленные каналы — использовать сжатие или бинарную сериализацию (в будущем).

---

## Заключение

Протокол передачи событий через `dev/<SN>/evt` обеспечивает:
- Единый формат данных
- Гибкость за счёт кодирования типов
- Поддержку множества сценариев (от датчиков до системного мониторинга)
- Совместимость с MQTT 5 и User Properties

Данный подход позволяет централизованно обрабатывать события от тысяч устройств в облаке.
