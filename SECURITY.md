# Security Policy

## Сообщить об уязвимости

**Не открывайте публичные GitHub Issues для сообщений о безопасности.**

Если вы обнаружили уязвимость, пожалуйста, сообщите о ней по электронной почте:

**info@platerra.ru**

Укажите в письме:
- Описание уязвимости
- Шаги воспроизведения
- Потенциальное влияние
- (опционально) Предлагаемое исправление

Мы постараемся ответить в течение 72 часов и согласовать план исправления.

## Политика по приватным ключам и сертификатам

**Никогда не коммитьте в репозиторий:**

- `*.pem` — сертификаты и ключи в PEM‑формате
- `*.key` — приватные ключи
- `*.pfx` — сертификаты в формате PKCS#12
- `.env` — файлы с переменными окружения и секретами
- Любые токены, пароли, API‑ключи

Сертификаты для mTLS должны передаваться через bind‑mount в Docker Compose.
Секреты — только через переменные окружения или `.env`‑файл (он перечислен в `.gitignore`).

Подробнее о настройке сертификатов: [`device-emulator/README.md`](device-emulator/README.md).
